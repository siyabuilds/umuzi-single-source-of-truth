### Team Boards  
Each Pillar is going to have 1 main board.  
Your team's work will be a group on the pillar most appropriate's main board.  
For example: For pillar 'Community', we'll have a General Board, with groups:  
- 'Hiring'  
- 'Staff Onboarding'  
- 'Alumni Council'  

Add a column for 'Audience' tag.  
List of Audience tags is:  
- 'Team'  
- 'Alumni'  
- 'Learners'  
- 'Partners'  
- 'Applicants'  

This dropdown needs to work for the whole board.  
Events - even if the task is 'Ask Sheena something' in the 'Events' group. Events need the tag 'Team', 'Alumni', and 'Learners'.  

We can now add filters for these groups 😄  
So for example, if you're the new HR manager and only want to check 'Team Recruitment', you can click on the 'Hiring and Recruitment' section below the main board's heading.  

As you can see in this screenshot above, I have selected the view 'Hiring and Recruiting' and I have set up filters:  
> Where group is 'Team Recruitment' OR 'Team Onboarding'.  

You can set as many filters as your needs require. You then save this filter to the board. Now, if you want to check on only the hiring and recruitment part within community, you've saved your filters to a new view. (A view is just a saved, filtered version of the main board which you can give its own name, kinda like a view on Airtable.)  

Each view that exists for your pillar's board will exist under the board's heading in a tab style format.
